📜 Syarat & Ketentuan Lisensi

Pembaruan Terakhir: Juli 2026

Terima kasih telah mempercayakan iMersTeleship sebagai sistem toko digital Telegram Anda. Dengan melakukan pembelian, mengunduh, menginstal, atau menggunakan source code ini, Anda dianggap telah membaca, memahami, dan menyetujui seluruh syarat serta ketentuan lisensi berikut.

1 Sistem Jual Putus Berbasis PHP & cPanel

Produk ini dijual dengan sistem jual putus berupa source code PHP, file dashboard admin, file webhook Telegram, struktur database, serta aset pendukung yang diperlukan untuk menjalankan sistem toko digital berbasis Telegram di hosting cPanel. Kami memastikan versi bawaan yang diberikan telah diuji pada lingkungan standar hosting PHP dan dapat berjalan sesuai fungsi utama sistem.

2 Kebutuhan Hosting, Domain, dan Server

Sistem ini membutuhkan hosting yang mendukung PHP, database yang sesuai dengan paket source code, SSL aktif, dan akses cPanel atau file manager untuk upload file. Kualitas hosting, limit server, pengaturan SSL, konfigurasi domain, proteksi firewall, atau pembatasan dari provider hosting berada di luar kendali kami.

Rekomendasi: gunakan hosting cPanel yang stabil, domain aktif, SSL HTTPS, dan pastikan URL webhook dapat diakses publik agar bot Telegram dapat berkomunikasi dengan sistem.

3 Hak Penggunaan White-Label

Anda diperbolehkan menggunakan sistem ini dengan brand Anda sendiri, termasuk mengubah logo, nama sistem, nama bot, foto profil bot, copywriting pesan, tampilan sales page, produk, harga, rekening pembayaran, QRIS, domain, dan identitas toko Anda sendiri. Anda juga bebas menentukan strategi harga jual produk digital yang dijual melalui bot Anda.

4 Modifikasi Source Code

Anda diperbolehkan memodifikasi source code sesuai kebutuhan dan kemampuan teknis Anda. Namun, apabila terjadi error, bug, kerusakan tampilan, kegagalan webhook, gangguan checkout, atau sistem berhenti berjalan akibat modifikasi kode, perubahan struktur database, perubahan alur logic, penghapusan file, kesalahan upload, atau konfigurasi hosting yang tidak sesuai, maka hal tersebut sepenuhnya menjadi tanggung jawab Anda.

Kami tidak berkewajiban memperbaiki source code yang sudah diubah, dirombak, digabung dengan sistem lain, atau dimodifikasi oleh pihak pembeli maupun pihak ketiga.

5 Batas Dukungan Teknis

Dukungan teknis hanya mencakup hal-hal berikut:
  • Panduan instalasi dasar sesuai dokumentasi resmi.
  • Bantuan pengecekan jika versi asli source code belum dimodifikasi dan terjadi kendala saat instalasi awal.
  • Laporan bug yang terbukti berasal dari versi bawaan source code, bukan karena perubahan kode, hosting, domain, atau konfigurasi pihak pembeli.

Dukungan tidak mencakup jasa custom fitur, edit tampilan khusus, integrasi payment gateway baru, migrasi hosting, debugging hasil modifikasi, atau konsultasi coding lanjutan. Kebutuhan tersebut dapat diajukan sebagai layanan custom berbayar jika tersedia.

6 Metode Pembayaran Manual

iMersTeleship mendukung sistem pembayaran manual seperti transfer bank, e-wallet, dan QRIS statis sesuai data yang Anda masukkan di dashboard admin. Sistem ini tidak menjamin, memverifikasi otomatis, atau bertanggung jawab atas validitas pembayaran di luar alur yang tersedia. Proses pengecekan, approval, refund, komplain pembayaran, dan layanan pelanggan akhir sepenuhnya menjadi tanggung jawab pemilik toko.

7 Keamanan Token, Admin, dan Data

Anda bertanggung jawab menjaga keamanan Bot Token Telegram, Admin Chat ID, username dan password dashboard, akses hosting, database, file konfigurasi, serta data pelanggan Anda. Jangan membagikan token bot, akses cPanel, atau kredensial admin kepada pihak yang tidak dipercaya.

Jika token bot atau akses admin pernah tersebar, segera lakukan penggantian token melalui BotFather, ubah password dashboard, dan cek ulang file konfigurasi di hosting Anda.

8 Larangan Distribusi Source Code / PLR / MRR

Anda dilarang keras menjual ulang, membagikan gratis, menyewakan, mengunggah ke marketplace source code, membocorkan, mendistribusikan, atau menjadikan source code mentah ini sebagai produk PLR, MRR, reseller source code, bundle source code, atau produk turunan sejenis.

Jika Anda ingin menjual sistem ini kepada klien, Anda hanya diperbolehkan bertindak sebagai penyedia jasa setup / agensi terima beres. Klien hanya boleh menerima sistem yang sudah terpasang dan siap pakai di hosting/domain mereka, bukan source code mentah untuk dijual ulang lagi.

9 Tanggung Jawab Terhadap End User

Kami sebagai developer asli tidak memiliki hubungan langsung, kewajiban layanan, tanggung jawab hukum, tanggung jawab finansial, atau kewajiban refund kepada pelanggan akhir dari toko Anda. Segala bentuk komplain, refund, dispute pembayaran, masalah akses produk, layanan pelanggan, dan konsekuensi transaksi sepenuhnya menjadi tanggung jawab Anda sebagai pemilik toko, brand, atau agensi yang menggunakan sistem ini.

10 Pembaruan dan Perbedaan Versi

Fitur, tampilan, struktur file, dan alur sistem dapat berubah mengikuti versi pengembangan terbaru. Pembeli hanya mendapatkan file sesuai paket dan versi yang dibeli pada saat transaksi, kecuali ada kebijakan update khusus yang dinyatakan secara terpisah. Jika Anda menggunakan versi yang sudah dimodifikasi, pembaruan dari versi terbaru mungkin tidak dapat langsung diterapkan tanpa penyesuaian teknis.
